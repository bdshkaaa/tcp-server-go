# Concurrent TCP Server in Go

Учебный проект, выполненный в рамках лабораторного практикума совместно с компанией **YADRO**.

## Цель
Реализовать конкурентный TCP-сервер, приближенный к промышленным требованиям:
- управление множественными соединениями
- ограничение ресурсов через Worker Pool
- корректное завершение без потери данных

## Архитектура
- **Worker Pool** на горутинах и каналах — фиксированное количество воркеров разбирают входящие соединения из очереди
- **Graceful Shutdown** — сервер перехватывает SIGINT/SIGTERM, перестаёт принимать новые соединения, даёт завершиться активным обработчикам
- **Таймауты** на чтение/запись — исключают зависшие соединения

## Стек
- Go (net/http, goroutines, channels, context, signal, sync)
- Тестирование: стандартная библиотека testing
